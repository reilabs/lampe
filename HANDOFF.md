# Handoff: PR #230 merkle_split Branch

## Initial Status (When Branch Started)

The `testing/Merkle` multi-package workspace was created to test lampe's ability to handle **Noir workspaces with multiple crates**. The workspace structure:

```
testing/Merkle/
├── hasher/           # Trait definition (no lampe/)
├── skyscraper/       # Hash implementation (has lampe/)
└── merkle/           # Merkle tree using skyscraper (has lampe/)
```

### Problems Found

1. **Test script didn't support workspaces** - `scripts/test.xsh` only handled single-crate projects

2. **Build failed with stack overflow**:
   ```
   error: Merkle-0.0.0/Spec.lean:84:2: maximum recursion depth has been reached
   ```

3. **Root cause**: The Noir code in `skyscraper/src/lib.nr` had a bug:
   ```noir
   // WRONG (was committed)
   s[0] + s[1]

   // CORRECT (should match MerkleFromScratch)
   s[0] + a
   ```
   This mismatch with the Lean spec `Ref.State.compress` caused `rfl` to fail.

---

## What Was Fixed

### 1. Workspace Test Support
Added to `scripts/test.xsh`:
- `is_workspace_test()` - Detects workspace vs single-crate projects
- `get_workspace_crates_with_lampe()` - Finds crates that need testing
- `run_workspace_test()` - Handles multi-crate extraction and building

### 2. Noir Bug Fix
User fixed `testing/Merkle/skyscraper/src/lib.nr`:
```noir
fn hash(a: Field, b: Field) -> Field {
    let s = permute::permute([a, b]);
    s[0] + a  // Fixed: was s[0] + s[1]
}
```

### 3. Increased maxRecDepth
Changed `Merkle-0.0.0/Spec.lean` from `maxRecDepth 2000` to `maxRecDepth 10000` for cross-crate trait resolution.

### 4. Regenerated Lampe Files
```bash
cd testing/Merkle/skyscraper && lampe
cd testing/Merkle/merkle && lampe
```

## Key Learnings

1. **Root library files should NOT have `-- Generated by lampe` headers** - By design, so users can customize them.

2. **Stack overflow in proofs** often means Noir code doesn't match Lean spec - Check they compute the same thing.

3. **Cross-crate proofs need higher `maxRecDepth`** (10000 vs 2000) due to deeper trait resolution.
